An analogy to describe JavaScript and its relationship to HTML and CSS.

JavaScript can be seen as the glue that binds HTML and CSS together. Just like glue, JavaScript's purpose is to connect them, and make them work together to create a user interface. HTML provides the structure and underlying content of the page, while CSS is the styling layer that defines how the page looks. JavaScript provides the interactivity, allowing the user to control and manipulate elements on the page. Without JavaScript, HTML and CSS would work together to create a static website, but with JavaScript, the website can become dynamic and interactive.

Explain control flow and loops using an example process from everyday life, for example, 'waking up' or 'brushing your teeth' (but not those).

Control flow and loops are important elements of programming that allow the computer to make decisions, execute certain tasks, and repeat processes. In an everyday example, think of doing laundry. When doing laundry, you must first make a decision about what laundry to do. This is like a control flow statement; it has logic to determine what tasks will be done. Next, you must complete multiple tasks, such as sorting clothes, adding detergent, and washing. This is like a loop, because it is a set of instructions that are repeated until complete. Finally, when all the tasks are completed, the process ends. This is an example of a control flow loop in everyday life.

Describe what the DOM is and an example of how you might interact with it.

The Document Object Model (DOM) is a platform- and language-independent interface that allows programs and scripts to interact with webpages and elements within them. It provides a tree-like structure of a webpage, allowing developers to access and update elements within it.
For example, if a developer wanted to use JavaScript to set the background color of a particular HTML element, they would first use the DOM to access it, then assign the color value. The example code below shows how this might look:
// Access the element
var el = document.getElementById('myElement');

// Set the background color
el.style.backgroundColor = '#FFF';

Explain the difference between accessing data from arrays and objects.

In programming, the two main ways to store data are in arrays and objects. Although both are collections of data, there are some distinct differences between them. Arrays are indexed collections where each item is stored in a specific location. Items can be accessed by their index, starting at 0 for the first item in the array. Arrays are best for data that needs to be ordered, like a list of products with their prices. Objects are unordered collections of data, where each item is stored with a key. Items can be accessed by their corresponding key, which can be any data type - from string to number. Objects are best for data that needs to be associated or grouped, like a list of products with their specs. Both arrays and objects are versatile and can be used to store any type of data. However, depending on the data you need to store, one may be more suitable than the other.

Explain what functions are and why they are helpful.

Functions are a way to organize code into easy-to-understand pieces that can be reused. Functions allow you to create code that is more efficient, scalable, and easier to debug. This reduces the amount of time needed to write and maintain code, making development faster and less expensive. Functions are used to separate a larger problem into smaller, more manageable pieces. This helps to make the code easier to read and maintain. It also allows for code reuse as the same code can be called multiple times without needing to rewrite it.
